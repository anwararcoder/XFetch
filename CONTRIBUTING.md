# Contributing to XFetch

First off, thank you for considering contributing to XFetch. It's people like you that make XFetch such a great tool.

## Setup

1. Fork and clone the repo
2. Run `npm install` to install dependencies
3. Create a branch for your PR

## Development Workflow

- Run `npm run dev` to build the library in watch mode.
- Write your feature and corresponding tests.
- We use `vitest` for tests. Run tests via `npm run test` or `npm run test:watch`.
- Make sure to use TypeScript effectively and avoid `any`.

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.
3. Ensure the test suite passes: `npm run release`.
4. You may merge the Pull Request in once you have the sign-off of at least one other developer.

## Core Maintainers

The core team will review your PR and guide you through any necessary changes.
